## Allowed dependencies

- `rule-engine-monorepo` (its packages, especially `rule-engine` core).
- Trading domain types (this is where it's specialized vs the agnostic engine).

## Forbidden dependencies

- L1 SDKs. The engine never directly talks to a broker — that's wired by the orchestrator.

## Architecture

- Same hexagonal pattern as the parent `rule-engine-monorepo`: domain definitions in `domain/`, validators in `application/`.
- Testkit provides realistic scenarios used by `MAGIC_POSITION/app/server` tests.
